# Правила безопасности OWASP (Skill)

1. **Управление секретами**:
   - НИКОГДА не захардкоживай пароли, API-ключи, токены или URI подключения к БД прямо в исходный код.
   - ВСЕГДА выноси их в переменные окружения (`.env`) и используй механизм управления секретами сервера (Secret Manager, Azure Key Vault).

2. **XSS (Cross-Site Scripting)**:
   - Всегда санитизируй (экранируй) пользовательский ввод при выводе данных в HTML. 
   - Полагайся на встроенные механизмы фреймворков (React, Vue, Razor экранируют строки автоматически). Запрещено использовать функции вроде `dangerouslySetInnerHTML` без 100% необходимости.
   
3. **CSRF (Cross-Site Request Forgery)**:
   - Убедись, что все мутирующие запросы (POST, PUT, DELETE, PATCH) защищены Anti-CSRF токенами или куки установлены с флагом `SameSite=Strict` или `Lax`.
   
4. **Санитизация и Валидация ввода**:
   - Не доверяй клиенту. Валидируй все данные, приходящие от пользователя (API endpoints, Forms), на стороне сервера ДО обработки логики. 
   - Используй строгую типизацию входных параметров API (Pydantic, Zod, FluentValidation).
