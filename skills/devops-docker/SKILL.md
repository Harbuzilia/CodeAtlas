# DevOps и Инфраструктура (Skill)

1. **Оптимизация Dockerfile**:
   - Всегда используй **Multi-stage builds** для компилируемых языков (C#, Go, TypeScript/Node.js). Образ с runtime не должен содержать исходников и тяжелых инструментов сборки.
   - Минимизируй количество промежуточных слоёв (объединяй несколько `RUN` команд в одну логическую цепочку).
   - Используй легковесные базовые образы (например, `alpine`, `distroless`, `slim`).

2. **Безопасность контейнеров**:
   - Не запускай приложения от лица пользователя `root` в контейнере. В конце Dockerfile создавай и переключайся на непривилегированного пользователя (например, `USER appuser`).

3. **Пайплайны (CI/CD)**:
   - Базовый пайплайн (GitHub Actions, GitLab CI) должен соблюдать логический порядок: Lint -> Test -> Build.
   - Запрещено деплоить или сшивать код, если падают тесты.
   
4. **Написание Bash-скриптов**:
   - Начало любого CI/CD shell-скрипта должно содержать `set -e` (для немедленного выхода при ошибке), а также желательно `set -u` и `set -o pipefail` для строгой проверки.
