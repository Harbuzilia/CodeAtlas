# Базы данных и SQL (Skill)

1. **Безопасность (SQL-инъекции)**:
   - НИКОГДА не используй конкатенацию строк для построения SQL-запросов (`"SELECT * FROM users WHERE id = " + id`).
   - ВСЕГДА используй параметризованные запросы (Parameterized queries).
   
2. **Производительность (ORM)**:
   - Внимательно следи за проблемой **`N+1` запросов**.
   - При работе с Entity Framework (C#), TypeORM/Prisma (TypeScript) или SQLAlchemy (Python) всегда используй жадную загрузку (`Include()`, `joinedload()`), если связанные сущности будут использоваться в цикле.
   
3. **Миграции схемы БД**:
   - Любое изменение схемы базы данных должно сопровождаться файлом миграции (Alembic, EF Migrations, TypeORM CLI и т.д.).
   - Не изменяй базу данных "на живую" через скрипты без контроля версий.

4. **Транзакции**:
   - При выполнении бизнес-логики, затрагивающей несколько таблиц (например, перевод денег: списание с одной, зачисление на другую), обязательно оборачивай операции в транзакции (Transaction Scope, Unit of Work).
